#### **当前实现（静态 fragments 库）**：

**✅ 推理流程（每天）**：
1. Kaggle 调用 `predict(test)` → test 包含当天的新数据（date_id 9021, 9022, ...）
2. `PredictionEngine._calculate_features_for_test()` → 把新 test 行添加到 `extended_data`
3. 计算新行的特征（Delta/Ratio）→ 需要用到历史数据（252 天均值）
4. 生成 embedding → 在**固定的 fragments.csv**（2652-8976）中搜索最相似片段
5. 返回 position → Kaggle 记录结果

**✅ 优点**：
- 快速：不需要每次重建 fragments（避免超时）
- 稳定：避免推理期逻辑变化

**⚠️ 缺点**：
- 无法利用推理期的新数据（12月16日后的市场状态不会加入 fragments）
- 匹配质量可能随时间推移下降（历史库与当前市场差异变大）

#### **可选实现（动态更新 fragments）**：

**方案A：每次推理时重建 fragments**（⚠️ 不推荐）
- 每次调用 `predict()` 时，用 `extended_data`（包含新 test 行）重新构建 fragments
- ❌ **严重问题**：每次重建耗时 > 5 分钟（超时）
- ❌ **数据泄漏风险**：可能意外使用未来数据

**方案B：定期批量更新 fragments**（⚠️ 实现复杂）
- 每周或每月重建一次 fragments（需要自定义调度逻辑）
- ❌ **Kaggle 限制**：不支持自定义调度，实现困难

**方案C：保持静态库，后期用 RL 优化**（✅ 推荐）
- 前期（12月-3月）：使用固定 fragments（2652-8976）
- 后期（3月-6月）：如果表现下降，启用 RL 动态调整策略
- ✅ **符合当前设计**：静态库 + 硬编码 7000 就是为 RL 准备的

#### **推荐策略**：
1. ✅ **第一次提交**：使用静态 fragments（当前实现）
2. ⏰ **监控推理表现**：观察 Sharpe Ratio 是否随时间下降
3. 🔄 **如需优化**：考虑 RL 或调整阈值（不是重建 fragments）

## 🎯 核心思路

## 📊 完整预测流程

### 步骤1：计算当前市场状态特征（Feature Engineering）

对当前日期 `date_id = X`，计算 **5个特征**：
```
特征计算规则：
- 差分 (Delta) = 当前值 - 昨日值 (一阶差分)
- 比率 (Ratio) = 5日均值 / 过去252日均值
```

**5个特征**：
- `E1_delta`: 宏观经济特征差分 (当前值 - 昨日值)
- `S1_delta`: 市场情绪特征差分 (当前值 - 昨日值)
- `V1_delta`: 波动率特征差分 (当前值 - 昨日值)
- `risk_free_rate_ratio`: 无风险利率比率 (5日均值 / 过去252日均值)
- `market_forward_excess_returns_delta`: 市场超额收益差分 (当前值 - 昨日值)

**关键点**：
- 使用**完整历史数据**回溯计算252日均值（确保有足够历史）
- 全局 ffill + bfill 处理缺失值

---

### 步骤2：生成Embedding向量

将5个特征比率转换为**5维embedding向量**：

```python
features = [

    V1_delta,

    market_forward_excess_returns_delta

embedding = StandardScaler.transform(features)  # 标准化
```

**输出**：`[5维向量]`，用于相似度搜索

```python
# 当前实现：动态过滤（预测 date_id=X 时，只用 end_date_id < X 的片段）
```

**搜索方法**：

- 找到**top-1**最相似的历史片段

**历史片段库**：

- **数据保留范围**：date_id >= 2400（约 6621 行）
- **片段构建起点**：date_id 2652（= 2400 + 252）

### 步骤5：计算Position（仓位）

当前实现是一个**单日决策版本**：每天根据当日的 `expected_excess_returns` 直接给出当日仓位。

#### 5.1 单日决策（当前实现）
target date = a
1. 计算 [a-4, a] 的 5 天特征
2. 生成 embedding，搜索历史中最相似的 [b-4, b]
3. 查看历史片段 [b-4, b] 之后的 **20 天平均超额收益**（`future_window=20`，可调参数）
4. 用这个 20 天平均作为 a 日的预期超额收益
5. 根据预期计算 a 日的 position
6. **次日（a+1）重复步骤 1-5，重新决策**（逐日独立决策，不持仓）

**当前实现特点**：
- ✅ 每天都重新计算 position（灵活调仓）
- ❌ 预测窗口（20天）与决策频率（1天）不匹配
- ❌ 交易成本可能较高（频繁换仓）

根据预期超额收益，使用**阈值判断**：

```python
# 当前代码实现的阈值（TODO-6：可通过回测优化）
if expected_excess_returns > 0.01:      # 正收益阈值
    position = 2.0  # 满仓做多
elif expected_excess_returns < -0.01:   # 负收益阈值
    position = 0.0  # 空仓
else:
    position = 1.0  # 中性持仓
```

**Position含义**：
- `0.0`: 空仓（全部持有无风险资产）
- `1.0`: 中性持仓（50%股票 + 50%无风险资产）
- `2.0`: 满仓做多（100%股票）

#### 5.2 长期持有策略（TODO-3：未来版本）

**目标设计**：
- 假设当前时间点 a，预测的 position 为 2，则连续 20 个交易日维持这个 position
- 20 个交易日后（时间点 a+20），重新计算并更新 position

**反转调仓机制**：
- 在持仓期间 [a+1, a+19]，持续监控每日预测信号
- 如果累计出现反向信号（position=0 或 1）超过 N 次（例如 2 次），则提前调仓
- 举例：当前 position=2，若连续 2 天预测出 position=0，立即调整为 0

**实现优先级**：

- 如果改为 1 天预测（TODO-2），则此逻辑不需要
