# 数据处理方式
来源：旧版 `预测逻辑说明.md` 和 `generate_fragments.py`。
## 核心规则
| 项目 | 规则 |
|---|---|
| 排序 | 按日期升序，重置索引 |
| 缺失值 | 只做 `ffill()`，禁止全局 `bfill()`，避免 ETF 上市前价格被未来数据倒灌 |
| 历史窗口 | 保留完整历史，用 252 日窗口计算状态 |
| market excess baseline | 默认使用 `yield_3m` 转日频 risk-free rate；rolling 5 年均值只作为 fallback |
| market excess MAD 窗口 | 1260 个交易日，约 5 年 |
| 短窗口 | 5 日 |
| 未来窗口 | 20 日 |
| 片段步长 | 5 日 |
| 标准化 | 用历史片段拟合 `StandardScaler`，再生成 embedding |
## 当前产物
| 文件 | 内容 |
|---|---|
| `data/processed/market_state.csv` | 每日市场状态表 |

## 特征变换
| 变换 | 计算方式 | 用途 |
|---|---|---|
| `delta` | `当前值 - 昨日值` | VIX 等日频一阶变化 |
| `ratio` | `5日均值 / 252日均值` | 价格位置、Treasury yield、CPI 相对水平 |
| `return` | `price_t / price_{t-1} - 1` | 资产日收益，不隐式填充缺失价格 |
| `zscore` | `(当前值 - 252日均值) / 252日标准差` | 接近 0 或低频变量的相对位置 |
| `delta` | `lagged_market_forward_excess_returns_t - lagged_market_forward_excess_returns_{t-1}` | target-like 字段的可观测变化 |
## P0 embedding
| 模块 | 字段 |
|---|---|
| 异常市场收益 | `lagged_market_forward_excess_returns_delta` |
| 波动率 | `VIX` 的 `delta` |
| 价格位置 | `SPY` 的 `ratio` |
| 利率 | 一个 Treasury yield：`yield_3m_ratio`，作为旧版短端利率代理 |
| 宏观 | `cpi_ratio`，作为 Kaggle `E1` 的真实代理 |
## fpR 标签
| 字段 | 计算方式 | 用途 |
|---|---|---|
| `future_20d_excess_returns` | 未来 20 行 `market_forward_excess_returns` 均值 | 旧版主预测标签 |
| `future_20d_volatility` | 未来 20 行 `market_forward_excess_returns` 标准差 | 风险诊断 |
| `fpR_20d_*_return` | 各资产未来 20 日复合收益 | 后续组合选择，不替代主标签 |
## market_forward_excess_returns
```text
forward_returns_t = SPY_{t+1} / SPY_t - 1
daily_rfr_t = (1 + yield_3m_t / 100)^(1 / 252) - 1
market_forward_excess_returns_t = forward_returns_t - daily_rfr_t
market_forward_excess_returns_t
  = winsorize_by_rolling_MAD(market_forward_excess_returns_t, criterion=4)
lagged_market_forward_excess_returns_t
  = market_forward_excess_returns_{t-1}
```
`yield_3m` 是年化百分比。
例如 `4.5` 表示 `4.5%`。
## 片段定义
```text
[ start_date ........ end_date ] [ future window ........ ]
          fpL                         fpR
   可观察市场状态                 后续真实资产表现
```
只有 `fpR` 已完整出现的历史片段才能进入可搜索历史库。
